# 1️⃣ O mercado de cardápio digital hoje

Os concorrentes seguem quase sempre **o mesmo modelo**.

## Funcionalidades padrão do mercado

Quase todos oferecem:

- Cardápio digital com QR Code
    
- Cadastro de produtos
    
- Categorias
    
- Atualização de preços
    
- Link público do cardápio
    
- Pedidos online
    
- Painel simples
    

Exemplo claro: plataformas como **Cardápio Tech**, **Virtual Cardápio** e outras fazem exatamente isso. ([Cardápio Technology](https://cardapiotech.com.br/?utm_source=chatgpt.com "Cardápio Technology - Cardápio Digital via QR Code | Modernize seu Restaurante"))

Alguns adicionam:

- QR Code por mesa
    
- pedidos online
    
- relatórios
    
- kitchen display (tela da cozinha)

Eles focam em **3 dores reais** do restaurante.

### 1️⃣ Atualizar preço rápido

Sem reimprimir cardápio.

### 2️⃣ Receber pedidos digitais

Direto da mesa ou online.

### 3️⃣ Reduzir custo

Comparado ao iFood.

Isso é exatamente o discurso deles:

# 5️⃣ Outro problema real do mercado

Muitos clientes **odeiam QR Code puro**.

Alguns comentários mostram isso claramente:

- clientes reclamam de usar celular no restaurante
    
- preferem cardápio físico
    
- acham ruim quando não dá para pedir direto ([Reddit](https://www.reddit.com/r/empreendedorismo/comments/1nq6l8l?utm_source=chatgpt.com "Vale a pena usar cardápio por qr code em restaurante?"))
    

💡 Ou seja:

QR Code **sozinho não resolve**.

O sistema precisa permitir:

```
ver cardápio
pedir direto
```

Isso melhora a experiência.
